# ULP Monitoring Dashboard Concept

Based on the analysis of `ulp-data-dharmais.csv`, the following dashboard concept is proposed to monitor procurement requests (Permohonan Pengadaan).

## 1. Data Overview
The dataset contains procurement request records with the following key dimensions and metrics:
- **Temporal**: Dates for DPP receipt (`Tanggal Diterima DPP`), End User letters, and DPP dates.
- **Organizational**: `Unit Kerja` (Work Unit), `Nama PPK` (Commitment Making Official).
- **Financial**: `Pagu Anggaran` (Budget Ceiling - both Active and Non-Active), `HPS` (Owner Estimate - note: appears to contain codes like B24, B25 in sample, potentially requiring data cleaning or mapping if numeric analysis is needed).
- **Categorical/Descriptive**: `Nama Paket PBJ` (Package Name), `Uraian MAK` (Budget Account Description), `Hal` (Subject).
- **Status/Process**: `Ka.ULP` (Current Status e.g., "Selesai dilaporkan", "Kaji Ulang PPK"), `Pokja Belanja Modal`, `Pokja Belanja OPRS`.

## 2. Proposed Dashboard Layout & Components

### A. Executive Summary (KPI Cards)
High-level metrics to give an instant view of the procurement landscape.

1.  **Total Requests (Jumlah Permohonan)**
    *   **Data**: Count of `No. Agenda` or rows.
    *   **Purpose**: Volume of procurement requests.
2.  **Total Active Budget (Total Pagu Anggaran Aktif)**
    *   **Data**: Sum of `Pagu Anggaran (Aktif)`.
    *   **Format**: Currency (IDR).
    *   **Purpose**: Total financial value of active procurement plans.
3.  **Pending/Revision Requests**
    *   **Data**: Count of rows where `Ka.ULP` is "Kaji Ulang PPK" or other non-final statuses.
    *   **Purpose**: Highlight bottlenecks or items requiring attention.
4.  **Completed/Reported Requests**
    *   **Data**: Count of rows where `Ka.ULP` is "Selesai dilaporkan".
    *   **Purpose**: Track completion progress.

### B. Charts & Visualizations

#### 1. Budget Distribution by Work Unit (Unit Kerja)
*   **Type**: Bar Chart (Horizontal).
*   **X-Axis**: `Pagu Anggaran (Aktif)`.
*   **Y-Axis**: `Unit Kerja` (e.g., TRT, IGZ, IKL).
*   **Insight**: Identifies which departments have the largest procurement spending plans.

#### 2. Request Volume by PPK (Pejabat Pembuat Komitmen)
*   **Type**: Column Chart or Donut Chart.
*   **Metric**: Count of Requests.
*   **Dimension**: `Nama PPK`.
*   **Insight**: Shows workload distribution among different PPKs (e.g., PPK SDM dan Gaji, PPK Jasa Konstruksi).

#### 3. Status Composition
*   **Type**: Pie Chart / Donut Chart.
*   **Dimension**: `Ka.ULP` (Status).
*   **Metric**: Count of Requests.
*   **Insight**: Visualizes the proportion of requests that are completed vs. those under review or revision.

#### 4. Monthly Request Trend
*   **Type**: Line or Area Chart.
*   **X-Axis**: Month of `Tanggal Diterima DPP`.
*   **Y-Axis**: Count of Requests.
*   **Insight**: Tracks the influx of procurement requests over time to identify peak periods.

#### 5. Budget Category Breakdown (MAK)
*   **Type**: Tree Map or Bar Chart.
*   **Dimension**: `Uraian MAK (Rinci)` (e.g., Belanja Jasa Keamanan, Belanja Cleaning Service).
*   **Metric**: `Pagu Anggaran (Aktif)`.
*   **Insight**: Details what the budget is actually being spent on.

### C. Detailed Data View (Table)
A tabular view for deep-diving into specific records.
*   **Columns**:
    *   No. Agenda
    *   Tanggal Terima
    *   Unit Kerja
    *   Nama Paket PBJ
    *   Pagu Anggaran (Aktif)
    *   Status (Ka.ULP)
*   **Features**:
    *   Search by Package Name or Agenda Number.
    *   Sortable columns (e.g., by Budget highest to lowest).
    *   Row coloring based on Status (e.g., Red for "Kaji Ulang", Green for "Selesai").

## 3. Filters & Slicers
To allow users to interact with the data:
*   **Date Range**: Filter by `Tanggal Diterima DPP` (Start Date - End Date).
*   **Unit Kerja**: Dropdown to select specific departments.
*   **PPK**: Dropdown to filter by specific Commitment Making Official.
*   **Status**: Checkbox to toggle between Completed, Revision, etc.

## 4. Data Quality & Transformation Notes
*   **Currency Parsing**: `Pagu Anggaran` columns contain commas (e.g., "6,876,713,444") and must be parsed as numbers.
*   **Date Parsing**: Dates are in format "dd-Mon-yy" (e.g., "20-Nov-24") and need conversion to Date objects.
*   **HPS Column**: The sample data shows codes (e.g., "B24") in the HPS column for some rows, while others might have values. Usage of this column for financial analysis should be verified.
